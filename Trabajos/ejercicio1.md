Contenido más adecuado para los siguientes servidores web.

**Apache:** Servidor web que implementa el protocolo http, es el servidor web más usado. Los archivos son dinámicos y estáticos.

**Nginx:** Servidor web de proxy inverso, usado principalmente para protocolos de correo eléctronico IMAP y POP3. Los archivos son estáticos, soporta http y http2 sobre ssl

**Thttpd:** Servidor web usado para protocolo http, se caracteriza por ser simple pequeño portatil rápido y seguro, ya que usa los requerimientos mínimos de un servidor http. Por tanto es ideal para servir grandes volúmenes de información estática.

**Cherokee:** Servidor web escrito en C. Rápido y completamente funcional. Puede usarse como sistema embebido y soporta complementos como FastCGI, SCGI, PHP, CGI, SSI, SSL/TLS.

**Node.js:** Entorno para la capa de servidor en tiempo de ejecución, no se limita sólo a ser un servidor. Basado en el lenguaje ECMAScript. Usa una arquitectura orientada a eventos. Creado para que fuera útil en la creación de programas de red escalables como son los servidores web.
